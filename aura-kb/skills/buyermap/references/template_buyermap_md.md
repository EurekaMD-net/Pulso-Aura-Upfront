## Reglas de formato del archivo

- **Markdown limpio**: encabezados jerárquicos H1 / H2 / H3 consistentes.
- **Tablas markdown legibles**: anchos balanceados, sin celdas excesivamente largas.
- **Sin emojis decorativos** salvo los semáforos funcionales (✅ ❌) en implicación estratégica.
- **Negritas para términos técnicos clave** la primera vez que aparecen.
- **Cursivas para citas o conceptos en inglés**.
- **Listas con bullets para enumeraciones**, no para todo.
- **No usar imágenes** (el archivo debe ser portable y procesable por otros LLMs).
- **Densidad informativa alta**: cada celda de tabla y cada bullet debe tener contenido sustantivo, no padding.

## Nombrado del archivo

`BUYERMAP_[Marca limpia sin espacios]_[Mercado abreviado]_[YYYY-MM-DD].md`

Ejemplos:
- `BUYERMAP_PalmolivePremium_Mexico_2026-03-15.md`
- `BUYERMAP_Tubi_LATAM_2026-06-22.md`
- `BUYERMAP_GranViaRoosevelt_Guatemala_2026-08-01.md`
